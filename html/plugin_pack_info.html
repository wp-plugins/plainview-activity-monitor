The Plainview Activity Monitor Plugin Pack is an actively maintained collection of plugins that expand the functionality of the Activity Monitor.

For a complete list of features and more information, see Plainview Activity Monitor Pack's page on the web.

Plugin Description
Bruteforce Detect

This plugin for the Activity Monitor detects bruteforce logins from one or several IPs and fires a bruteforce_detect action for each IP when the criteria have been fulfilled.

This action can then be used by a plugin that can ban the IP address(es).

List of failed logins grouped by username / password / user agent
Grouping settings for detection sensitivity and limits

Content Watch

Monitors posts and pages for specific phrases and fires an action when the content is detected.

Use in conjunction with the Send To E-mail to inform administrators that the user has been writing things it shouldn’t have.

Phrases to monitor
Test the content watch using a specific post.

IP Too Often

Monitor IP addresses for recurring actions. Choose which actions to monitor and when the limits have been surpassed, fire a unique ip_too_often_xxxx action.

An overview of the available watches (settings)
Editing what to monitor and how often

Login Failed Username

Specify a list of usernames that may not fail when logging in. If the username is found, the login_failed_username action is fired.

The username need not exist. If your main admin user is called admin953551, then you can specify the username admin in order to be informed of someone trying to break in by guessing.

Usernames to monitor for login failures

Send To E-mail

Send an e-mail to various recipients when actions are fired. In each drop there are settings for e-mail sender and recipients, e-mail subject and text body. The subject and body can be modified with keywords that are replaced with information from the action.

Different hook groups can be e-mailed to different recipients
Drop settings: hooks, recipients, subject and text to send

Send To Exec

When specific actions are encountered, send the action to an executable command. The command can be modified with information from the action by using keywords that are automatically replaced.

Different hook groups can be sent to different executable files
Set up what is to be sent to the executable file
